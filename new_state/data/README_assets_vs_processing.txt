素材 vs Processing 生成 — 简要分析
====================================

一、当前做法
-----------
• 鞭炮 (firecracker)：优先从 ../assets/firecracker/firecracker_1.png 等加载；
  若文件缺失，则用 Processing 绘制占位（小椭圆模拟鞭炮外形），程序照常运行。
• 柱子 (pillar)：优先从 ../assets/pillar/pillar_1.png 等加载；
  若文件缺失，则用 Processing 绘制占位（矩形+顶盖），程序照常运行。
• 烟花 (firework)：完全由 Processing 绘制，无外部素材。从画面下方发射，在天空 Y 50~200 爆炸。

二、用素材好还是 Processing 生成好？
-----------------------------------
【适合用素材的情况】
  - 需要写实、统一风格（如真实鞭炮/柱子照片、手绘资源）。
  - 美术已有现成图，或希望后期只换图不改代码。
  - 表现复杂细节（纹理、光影）时，素材通常更省事、效果更可控。

【适合用 Processing 生成的情况】
  - 无现成素材、或希望零依赖运行（不报 missing file）。
  - 需要程序化变化（如烟花随机颜色/粒子数、柱子高度变化）。
  - 占位/原型阶段，先保证逻辑和节奏正确，再替换成正式图。

三、建议
-------
• 鞭炮/柱子：保留「有素材用素材、没有则 Processing 占位」的折中方案；
  正式发布前若有了美术资源，把图放到 ../assets/firecracker/ 与 ../assets/pillar/ 即可自动替换。
• 烟花：已用 Processing 实现，风格可调（颜色、粒子数、爆炸高度在 BlessingConfig 中配置）；
  若以后需要更写实的烟花，可再增加「可选素材序列」作为补充，与当前粒子效果二选一或叠加。
